2-way Insertion Sort

Posted on 2020-01-01

数据结构

二路插入排序（C语言版）

1. 代码

  折半插入排序仅仅优化了插入排序的查找过程，减少了比较次数，而二路插入排序（2-way Insertion Sort）在折半插入排序的基础上，优化了插入过程，减少了移动次数。
  先约定一下记号，$A$为待排序数组，$B$为新开辟数组，$C$为排序完成的数组，$a$为$A$的第一个元素。
  基本想法是这样的：
  首先，在最终排序完成的数组$C$中，元素$a$将数组$C$分成两部分，$C = [C_1, a, C_2]$，其中$C_1$、$C_2$为有序数组，且$C_1$中元素小于$a$，$C_2$中元素大于等于$a$。（由于$a$是$A$的第一个元素，所以为了保持排序的稳定性，与$a$相等的元素会放到$C_2$中）
  因此，我们可以开辟$C_1$，$C_2$两个新数组，将小于$a$的元素放入$C_1$进行折半插入排序，将大于等于$a$的元素放入$C_2$进行折半插入排序。当元素插入$C_1$时，就不必移动$C_2$中的元素。这样，就减少了移动次数。
  在实际实现中，不必开辟两个数组来实现。我们可以用一个循环数组$B$来代替$C_1$,$C_2$。具体做法为：将$a$放到$B$的第一个位置，$a$之前的空间为$C_1$，之后为$C_2$。

2.总结

  • 注意二分查找数组的插入位置时，向右移动和向左移动的位置是不一样的
  • 插入排序时，第一个元素是给定的，所以这里$C_1$的第一个元素我只能单独处理的，不知道有没有更好的办法。
  • 虽然数组$B$是循环数组，但这里并没有体现出来，因为这里的求余操作$\%$没有什么用，所以仍有改进空间
  • 没能给出数组$C_1$，$C_2$统一的查找、插入方法，而是分开来做的。

2.问题

  • 为什么high + 1是记录插入位置？
  • 为什么L.r[0].key < L.r[m].key判断出插入点在高半区而不要加等号？
  • 为什么查找过程while循环条件是low <= high? 去掉等号会怎么样？

3.尝试解决

  • 为什么high + 1是记录插入位置?
    • 折半查找过程中倒数第二次折半后，只剩下$1$或$2$个可能位置
        在折半查找过程时，每查找一次，可能位置个数减半，具体来说：设目前位置个数为$L$，若$L$为偶数，则经过一次折半查找后的位置个数变为$L/2$或$L/2 - 1$，若$L$为奇数，则位置个数变为$\lfloor L/2 \rfloor$.
        且由$L = high - low + 1$可知循环条件low <= high等价于$L >= 1$.
        设$L$是倒数第二次减半的位置个数，则其必须满足循环条件，且下一次折半后不满足循环条件。
        若$L$是奇数，则$\begin{cases}L>=1 \\ \lfloor L/2 \rfloor < 1\end{cases}$,可得$L = 1$；
        若$L$是偶数，满足$\begin{cases}L>=1 \\ L/2 < 1 \ 或\ L/2 - 1 < 1\end{cases}$ $\Leftrightarrow$$\begin{cases}L>=1 \\ L < 4\end{cases}$, 可得$L = 2$.
    • 当$L = 1$时：
r[0] $\cdots$ r[m - 1] r[m] r[m + 1] $\cdots$
$\uparrow_{high}$
$\uparrow_{mid}$
$\uparrow_{low}$ $ $

$\underrightarrow{\text{若r[0] < r[m], high = mid-1}}$

r[0] $\cdots$ r[m - 1] r[m] r[m + 1] $\cdots$
$\uparrow_{high}$
$\uparrow_{mid}$
$\uparrow_{low}$ $ $

此时high < low，循环结束，由于r[0] < r[m]则插入位置应为high + 1。同样的，若r[0] >= r[m]则low = mid + 1，此时循环结束，则插入位置也应为high + 1.（注意：当r[0] = r[m]时，可以看出r[0]被排在r[m]后面）


当$L = 2$时：

r[0] $\cdots$ r[m - 1] r[m] r[m + 1] $\cdots$
$\uparrow_{high}$
$\uparrow_{mid}$
$\uparrow_{low}$ $ $

$\underrightarrow{\text{若r[0] < r[m], high = mid - 1}}$

r[0] $\cdots$ r[m - 1] r[m] r[m + 1] $\cdots$
$\uparrow_{high}$
$\uparrow_{mid}$
$\uparrow_{low}$ $ $

此时循环结束，插入位置应为high + 1。若r[0] >= r[m]则low = mid + 1，此时变成$L = 1$的情况。


以上就证明了插入位置确实是high + 1


2020.1.1 补充

  想到了更简单的理解：代码中可以看出，循环结束时，high指标的右边是所有数据中大于r[0]的，而low指标左边则是所有数据中小于等于r[0]的，所以插入位置可以为high + 1，也可以为low。

  进一步地，这段代码的想法是：将每个数据划到对应的部分（大于r[0]部分或者小于等于r[0]的部分）。因此while循环条件需要判断是否还有剩余的数据没有归类，剩余数据的下标范围是[low, high]。是否有剩余数据，等价于判断这个整数集合是否为空，也就等价于low <= high ?。

  这样来想，下面两个问题就很容易回答了。

  • 为什么L.r[0].key < L.r[m].key判断出插入点在高半区而不要加等号?
    通过对问题一的分析，我们可以看出，这是为了保持插入排序的稳定性

  • 为什么查找过程while循环条件是low <= high? 去掉等号会怎么样？
    若去掉等号，碰到high = low的情况就会导致循环结束，此时没有比较r[0]和r[m]，无法确定插入位置。

4.总结

初步理解了这个算法，但感觉会有更直观的理解方法，且$L$的意义说的不是很清楚。
